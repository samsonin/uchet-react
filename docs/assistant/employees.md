# Страница: Сотрудники

Маршрут в приложении: `/settings/employees`

## Когда использовать

Используй эту инструкцию, когда пользователь спрашивает:

- где посмотреть сотрудников организации;
- как найти сотрудника;
- как пригласить нового пользователя;
- как изменить должность сотрудника;
- как уволить или вернуть сотрудника;
- где настроить должности и права доступа;
- почему не получается удалить должность.

## Краткий ответ для пользователя

На странице `/settings/employees` можно управлять сотрудниками организации: искать сотрудников, назначать конкретным сотрудникам должности, менять их статус, добавлять новые должности и регулировать полномочия для каждой должности.

## Что можно сделать на странице

1. Найти сотрудника через строку поиска. Поиск работает по имени, телефону и e-mail.
2. Включить переключатель `включая уволенных`, чтобы увидеть не только работающих, но и уволенных сотрудников.
3. Нажать `Пригласить пользователя`, чтобы перейти на страницу приглашений `/settings/invites` и создать приглашение для нового сотрудника.
4. Изменить должность сотрудника в выпадающем списке в колонке `Должность`.
5. Переключить статус сотрудника: `Работает` или `Уволен`. Если сотрудник переведен в статус `Уволен`, он не сможет войти в приложение.
6. В блоке `Доступы должностей` настроить полномочия для каждой должности.
7. Нажать `Добавить должность`, чтобы создать новую должность и сразу выбрать для нее доступы.
8. Нажать иконку редактирования рядом с названием должности, чтобы изменить должность или ее доступы.
9. Удалить должность можно только если к ней не привязан ни один сотрудник. Если сотрудники есть, сначала переведите их на другую должность.

## Подсказки для ассистента

- Если пользователь уже находится на `/settings/employees`, не предлагай ему открыть эту же страницу или список сотрудников.
- Если пользователь хочет добавить сотрудника, объясняй, что новый пользователь подключается через приглашение на `/settings/invites`.
- Если пользователь хочет изменить доступы, направляй его к блоку `Доступы должностей`, а не к отдельному сотруднику: права задаются на уровне должности.
- Если пользователь не видит уволенного сотрудника, предложи включить `включая уволенных`.
- Если пользователь спрашивает про статус `Работает` / `Уволен`, объясни, что статус `Уволен` отключает сотруднику вход в приложение.
- Если не получается удалить должность, объясни, что должность нельзя удалить, пока есть сотрудники с этой должностью.
- Не обещай создание учетной записи сотрудника прямо на странице сотрудников: здесь есть переход к приглашениям.
- Не говори, кому доступен или недоступен этот раздел. Если пользователь уже находится на странице, значит, доступ у него есть.

## Пример ответа ассистента

На странице `Сотрудники` можно найти сотрудника по имени, телефону или e-mail, назначить конкретному сотруднику должность и переключить статус `Работает` / `Уволен`. Если выбрать `Уволен`, сотрудник больше не сможет войти в приложение. Здесь же можно добавить новые должности и настроить полномочия для каждой должности в блоке `Доступы должностей`. Чтобы добавить нового сотрудника, нажмите `Пригласить пользователя` и создайте приглашение на странице `/settings/invites`.
